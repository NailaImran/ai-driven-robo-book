## Part 1: Core Functionality (100 points)

### 1.1 Autonomous Navigation (20 points)

#### Excellent (18-20 points)
- ‚úÖ SLAM creates accurate, detailed maps
- ‚úÖ Localization error `<`5cm consistently
- ‚úÖ Navigates 10+ meters successfully in cluttered environment
- ‚úÖ Avoids both static and dynamic obstacles reliably
- ‚úÖ Reaches goals within 5cm accuracy `>`95% of the time
- ‚úÖ Replanning works smoothly when obstacles appear
- ‚úÖ No collisions during demonstrations

#### Good (14-17 points)
- ‚úÖ SLAM works but map has minor artifacts
- ‚úÖ Localization error `<`10cm
- ‚úÖ Navigates successfully in moderately complex environments
- ‚úÖ Avoids static obstacles reliably, struggles with dynamic ones
- ‚úÖ Goal accuracy within 10cm `>`80% of the time
- ‚úÖ Occasional replanning delays (`<`2 seconds)
- ‚úÖ Rare minor collisions (low speed, no damage)

#### Acceptable (10-13 points)
- ‚úÖ Basic SLAM functionality working
- ‚úÖ Localization error `<`20cm
- ‚úÖ Can navigate in simple environments (few obstacles)
- ‚úÖ Avoids large static obstacles
- ‚úÖ Reaches goal within 20cm `>`60% of the time
- ‚úÖ Replanning takes `>`2 seconds
- ‚úÖ Occasional collisions at low speed

#### Needs Improvement (`<`10 points)
- ‚ùå SLAM frequently fails or produces poor maps
- ‚ùå Localization error `>`20cm
- ‚ùå Cannot reliably navigate even simple paths
- ‚ùå Frequent collisions
- ‚ùå Success rate `<`60%

**Evaluation Criteria**:
- **Map Quality** (5 points): Accuracy, completeness, consistency
- **Path Planning** (5 points): Optimality, smoothness, safety margins
- **Obstacle Avoidance** (5 points): Static and dynamic obstacle handling
- **Success Rate** (5 points): Percentage of successful navigation attempts

### 1.2 Vision-Based Manipulation (25 points)

#### Excellent (23-25 points)
- ‚úÖ Detects 5+ object classes with `>`90% accuracy
- ‚úÖ Real-time performance (`>`15 FPS)
- ‚úÖ 6D pose estimation error `<`1cm translation, `<`3¬∞ rotation
- ‚úÖ Grasp success rate `>`85% over 20+ trials
- ‚úÖ Handles varied object sizes, shapes, orientations
- ‚úÖ Smooth, collision-free arm trajectories
- ‚úÖ Robust to partial occlusions and lighting changes

#### Good (19-22 points)
- ‚úÖ Detects 3-5 classes with `>`80% accuracy
- ‚úÖ Runs at `>`10 FPS
- ‚úÖ Pose estimation error `<`2cm, `<`5¬∞
- ‚úÖ Grasp success rate `>`70%
- ‚úÖ Works with standard objects (cubes, cylinders, simple shapes)
- ‚úÖ Occasional trajectory stuttering but no collisions
- ‚úÖ Works in controlled lighting

#### Acceptable (15-18 points)
- ‚úÖ Detects 2-3 classes with `>`70% accuracy
- ‚úÖ Runs at 5-10 FPS
- ‚úÖ Pose estimation error `<`5cm, `<`10¬∞
- ‚úÖ Grasp success rate `>`50%
- ‚úÖ Limited to specific object types
- ‚úÖ Some unsuccessful grasp attempts
- ‚úÖ Requires good lighting conditions

#### Needs Improvement (`<`15 points)
- ‚ùå Detection accuracy `<`70%
- ‚ùå Runs too slowly for real-time use
- ‚ùå Poor pose estimation
- ‚ùå Grasp success rate `<`50%
- ‚ùå Frequent collisions or failures

**Evaluation Criteria**:
- **Detection** (7 points): Accuracy, speed, robustness
- **Pose Estimation** (6 points): Translation and rotation errors
- **Grasp Planning** (6 points): Success rate, safety
- **Trajectory Execution** (6 points): Smoothness, collision avoidance

### 1.3 Bipedal Locomotion (20 points)

#### Excellent (18-20 points)
- ‚úÖ Stable walking on flat ground at variable speeds (0.1-1.0 m/s)
- ‚úÖ Climbs 3+ stairs (15-20cm height) without falls
- ‚úÖ Maintains balance during arm movements (reaching, carrying)
- ‚úÖ Recovers from small external perturbations
- ‚úÖ Smooth gait transitions (start, stop, turn)
- ‚úÖ No falls during 10+ minute continuous operation
- ‚úÖ Natural-looking gait pattern

#### Good (14-17 points)
- ‚úÖ Stable walking at fixed speed (~0.3 m/s)
- ‚úÖ Climbs 3 stairs with careful tuning
- ‚úÖ Can balance while stationary during manipulation
- ‚úÖ Occasional balance corrections visible
- ‚úÖ Some stuttering during gait transitions
- ‚úÖ No falls during 5-minute operation
- ‚úÖ Functional but robotic-looking gait

#### Acceptable (10-13 points)
- ‚úÖ Can walk short distances (2-5 meters) on flat ground
- ‚úÖ Climbs 1-2 stairs with difficulty
- ‚úÖ Must stop walking before manipulating
- ‚úÖ Visible instability, requires conservative speeds
- ‚úÖ Abrupt gait transitions
- ‚úÖ Occasional falls but recoverable

#### Needs Improvement (`<`10 points)
- ‚ùå Cannot walk reliably even on flat ground
- ‚ùå Cannot climb stairs
- ‚ùå Falls frequently
- ‚ùå Unstable during manipulation

**Evaluation Criteria**:
- **Walking Stability** (7 points): Balance, speed range, smoothness
- **Stair Climbing** (6 points): Number of steps, confidence, success rate
- **Balance During Tasks** (4 points): Manipulation while balancing
- **Robustness** (3 points): Recovery from perturbations

### 1.4 Conversational Interaction (20 points)

#### Excellent (18-20 points)
- ‚úÖ Recognizes 10+ command types with `>`95% accuracy
- ‚úÖ Handles natural language variations ("get the cup" vs "pick up the mug")
- ‚úÖ Context-aware responses (remembers previous commands)
- ‚úÖ Natural, clear TTS voice
- ‚úÖ Low latency (`<`1 second from voice to action)
- ‚úÖ Provides helpful feedback on task progress
- ‚úÖ Gracefully handles unrecognized commands

#### Good (14-17 points)
- ‚úÖ Recognizes 8-10 commands with `>`90% accuracy
- ‚úÖ Handles some variations in phrasing
- ‚úÖ Basic context awareness
- ‚úÖ Clear but robotic TTS
- ‚úÖ Response latency 1-2 seconds
- ‚úÖ Acknowledges commands but minimal feedback
- ‚úÖ Reports when command not understood

#### Acceptable (10-13 points)
- ‚úÖ Recognizes 6-8 commands with `>`80% accuracy
- ‚úÖ Requires specific phrasing
- ‚úÖ No context retention
- ‚úÖ Functional TTS (understandable but mechanical)
- ‚úÖ Response latency 2-3 seconds
- ‚úÖ Basic acknowledgments only
- ‚úÖ Sometimes fails silently on unrecognized commands

#### Needs Improvement (`<`10 points)
- ‚ùå Recognizes `<`6 commands
- ‚ùå Accuracy `<`80%
- ‚ùå Latency `>`3 seconds
- ‚ùå Poor TTS quality
- ‚ùå Frequent misunderstandings

**Evaluation Criteria**:
- **Speech Recognition** (6 points): Accuracy, speed, noise robustness
- **Language Understanding** (6 points): LLM integration, intent parsing
- **Response Quality** (4 points): TTS naturalness, feedback usefulness
- **Command Coverage** (4 points): Number and variety of supported commands

### 1.5 System Integration (15 points)

#### Excellent (14-15 points)
- ‚úÖ Single launch file starts entire system reliably
- ‚úÖ All modules communicate seamlessly
- ‚úÖ Graceful error handling (no crashes on failures)
- ‚úÖ System runs in real-time (latency `<`500ms)
- ‚úÖ Comprehensive diagnostics and monitoring
- ‚úÖ Can switch between simulation and hardware modes
- ‚úÖ Automatic recovery from common failures

#### Good (11-13 points)
- ‚úÖ Launch file works after minor manual steps
- ‚úÖ Modules communicate with occasional hiccups
- ‚úÖ Basic error handling (logs errors, may crash)
- ‚úÖ Mostly real-time with occasional lags
- ‚úÖ Basic status monitoring
- ‚úÖ Simulation mode fully functional
- ‚úÖ Manual recovery from failures

#### Acceptable (8-10 points)
- ‚úÖ Multiple launch files needed
- ‚úÖ Communication works but with noticeable delays
- ‚úÖ Minimal error handling
- ‚úÖ Latency sometimes `>`500ms
- ‚úÖ Limited monitoring
- ‚úÖ Works in simulation only

#### Needs Improvement (`<`8 points)
- ‚ùå Cannot launch system reliably
- ‚ùå Modules don't communicate properly
- ‚ùå Frequent crashes
- ‚ùå Unacceptable latency
- ‚ùå No monitoring or error handling

**Evaluation Criteria**:
- **Launch System** (4 points): Ease of startup, reliability
- **Communication** (4 points): ROS 2 topics/services/actions
- **Error Handling** (4 points): Graceful failures, recovery
- **Performance** (3 points): Real-time operation, resource usage

## Part 2: Code Quality (10 points)

### Excellent (9-10 points)
- ‚úÖ Modular, well-organized code structure
- ‚úÖ Follows ROS 2 best practices and conventions
- ‚úÖ Comprehensive docstrings for all functions/classes
- ‚úÖ Meaningful variable and function names
- ‚úÖ Consistent code style (formatted with black/clang-format)
- ‚úÖ No code duplication
- ‚úÖ Proper error handling throughout
- ‚úÖ Type hints (Python) or strong typing (C++)

### Good (7-8 points)
- ‚úÖ Mostly modular with some coupling
- ‚úÖ Generally follows conventions
- ‚úÖ Most functions have docstrings
- ‚úÖ Clear naming (with a few exceptions)
- ‚úÖ Mostly consistent style
- ‚úÖ Some code duplication
- ‚úÖ Error handling in critical sections

### Acceptable (5-6 points)
- ‚úÖ Basic organization
- ‚úÖ Some convention violations
- ‚úÖ Minimal documentation
- ‚úÖ Adequate naming
- ‚úÖ Inconsistent style
- ‚úÖ Moderate duplication
- ‚úÖ Basic error handling

### Needs Improvement (`<`5 points)
- ‚ùå Poor organization
- ‚ùå Doesn't follow conventions
- ‚ùå No documentation
- ‚ùå Unclear naming
- ‚ùå No consistent style
- ‚ùå Significant duplication

**Evaluation Checklist**:
- [ ] PEP 8 / Google C++ Style compliance
- [ ] All packages have package.xml with dependencies
- [ ] README explains code structure
- [ ] No hardcoded paths or credentials
- [ ] No commented-out code blocks
- [ ] Git history shows good commit practices

## Part 4: Bonus Features (up to +30 points)

### Multi-Modal Perception (+10 points)

**Full Credit (10 points)**:
- Implements sensor fusion (camera + LiDAR + IMU)
- Uses Extended Kalman Filter or Particle Filter
- Quantitatively demonstrates improved accuracy
- Handles sensor failures gracefully
- Documented in technical report

**Partial Credit (5-9 points)**:
- Basic sensor fusion
- Simple averaging or voting schemes
- Some improvement shown
- Limited failure handling

### Reinforcement Learning (+15 points)

**Full Credit (15 points)**:
- Trains RL policy from scratch for meaningful subtask
- Uses modern RL framework (Isaac Gym, Stable Baselines3)
- Shows learning curves and ablation studies
- Demonstrates sim-to-real transfer
- Policy outperforms hand-coded baseline
- Thoroughly documented

**Partial Credit (8-14 points)**:
- Uses pre-trained policy with fine-tuning (8-10 pts)
- Training in simulation only, no transfer (10-12 pts)
- Comparable performance to baseline (12-14 pts)

### Human-Robot Collaboration (+10 points)

**Full Credit (10 points)**:
- Detects and tracks humans in environment
- Recognizes gestures (pointing, waving, etc.)
- Responds to non-verbal cues
- Maintains safe distances
- Hands objects to humans safely
- Demonstrated in realistic scenarios

**Partial Credit (5-9 points)**:
- Basic human detection
- Limited gesture recognition
- Static interaction only

### Dynamic Environments (+10 points)

**Full Credit (10 points)**:
- Navigates around 3+ moving obstacles
- Real-time replanning (`<`500ms)
- Predictive obstacle avoidance
- Tested in challenging scenarios
- Success rate `>`80%

**Partial Credit (5-9 points)**:
- Handles 1-2 moving obstacles
- Replanning takes 1-2 seconds
- Reactive avoidance only

### Real Hardware Deployment (+20 points)

**Full Credit (20 points)**:
- All core features work on physical robot
- Comprehensive sim-to-real transfer analysis
- Video demonstration on hardware
- Handles real-world uncertainties
- Documented calibration process
- Compare hardware vs simulation results

**Partial Credit (10-19 points)**:
- Most features work on hardware (10-12 pts)
- Limited transfer analysis (13-15 pts)
- Some features degraded on hardware (16-19 pts)

## Common Deductions

### Functionality Issues
- Missing required capability: **-20 to -25 points** each
- Feature works but unreliably: **-5 to -10 points**
- Hardcoded/cheated solution: **-15 to -20 points**

### Code Quality Issues
- No documentation: **-3 points**
- Hardcoded paths: **-2 points**
- Poor organization: **-3 points**
- Doesn't follow conventions: **-2 points**

### Documentation Issues
- Missing section in report: **-2 points** each
- Poor video quality: **-2 points**
- No README: **-3 points**
- Plagiarism: **0 points** (automatic failure)

## Grade Appeals

If you believe your grade is incorrect:

1. **Review rubric carefully** - most disputes come from misunderstanding criteria
2. **Check for partial credit** - you may have received points you didn't notice
3. **Prepare evidence** - video timestamps, code references, metrics
4. **Submit written appeal** within 1 week of grade release
5. **Meet with instructor** to discuss

**Note**: Grade appeals may result in lower scores if errors in your favor are discovered.

---

## Tips for Maximizing Your Score

1. **Start with core requirements** - don't attempt bonus features until all base functionality works
2. **Test thoroughly** - record metrics from 10+ trials, not just one successful run
3. **Document as you go** - don't wait until the end
4. **Follow the rubric** - check each criterion specifically
5. **Proofread everything** - spelling/grammar errors hurt professionalism
6. **Record backup videos** - if one demo fails, you have alternatives
7. **Ask for feedback early** - office hours are your friend
